# Radiant Dental Website

A professional, responsive dental website built with HTML, CSS, and JavaScript, designed to showcase dental services, introduce the team, and facilitate patient appointments.

## Description

Radiant Dental is a modern website for a dental practice, providing an online presence that highlights expert dental care services. The site aims to build trust with potential patients by presenting information about services, the experienced team of dentists, and ways to get in touch. It features a clean, user-friendly design optimized for both desktop and mobile devices.

## Features

### Core Functionality
- **Responsive Design**: Fully responsive layout that adapts to various screen sizes, ensuring a great user experience on desktops, tablets, and smartphones.

- **Navigation**: Sticky header with smooth scrolling navigation to different sections of the site.

- **Contact Integration**: Header includes email and phone contact links, with a prominent "Book Appointment" call-to-action button.

- **Social Media Links**: Integrated social media icons for Facebook, Instagram, Twitter, and YouTube in the header and footer.

### Sections
- **Hero Section**: Eye-catching landing area with a background image, welcoming message, tagline, and an email form for requesting a callback.
- **Services Section**: Showcases six key dental services:
  - Root Canal: Precision care to relieve pain and save teeth.
  - Alignment Teeth: Expert orthodontics for a perfect smile.
  - Cosmetic Teeth: Aesthetic enhancements for stunning results.
  - Oral Hygiene: Daily care tips and professional hygiene services.
  - Live Advisory: Real-time guidance and support.
  - Cavity Inspection: Early detection and preventive care.

- **About Us Section**: Describes the clinic's commitment to dental health, personalized treatments, and patient comfort.
- **Doctor Profiles**: Introduces the expert team with photos and social links:
  - Howard Holmes
  - Ella Thompson
  - Vincent Cooper
  - Danielle Bryant

- **Call-to-Action (CTA)**: Encourages booking appointments with an inviting banner and button.

- **Blog Section**: Features the latest news and articles, including three sample blog posts with images, dates, and summaries.

- **Footer**: Comprehensive footer with clinic description, quick links, service list, contact details (address, phone, email), operating hours, copyright notice, and social media links.
- **Back to Top Button**: Convenient navigation to return to the top of the page.

### Interactive Elements
- **JavaScript Enhancements**: Includes scripts for dynamic behavior such as menu toggling, smooth scrolling, and form handling.

- **Icons**: Uses Ionicons for consistent, professional iconography throughout the site.

- **Forms**: Email input form in the hero section for lead generation.

## Technologies Used

- **HTML5**: Semantic structure for accessibility and SEO.
- **CSS3**: Custom styling with responsive design, including Flexbox and Grid for layout.
- **JavaScript (ES6+)**: For interactivity, including event handling and DOM manipulation.
- **Google Fonts**: Poppins and Roboto fonts for modern typography.
- **Ionicons**: Icon library for UI elements.
- **Images and Assets**: Custom images for banners, icons, doctor photos, and blog posts stored in the `Assets/` folder.

## Getting Started

### Prerequisites

- A modern web browser (e.g., Chrome, Firefox, Safari, Edge) to view the website.
- No server-side dependencies; it's a static site.

### Customization

- **Content**: Edit text, images, and links in `index.html`.
- **Styling**: Modify `style.css` to change colors, fonts, or layouts.
- **Functionality**: Update `script.js` for additional interactive features.
- **Assets**: Replace images in the `Assets/` folder with your own (ensure consistent naming and formats).

## Contributing

We welcome contributions to improve the website! Here's how you can help:

1. **Fork the Repository**: Create your own fork on GitHub.
2. **Create a Feature Branch**: `git checkout -b feature/your-feature-name`
3. **Make Changes**: Edit the relevant files.
4. **Test Locally**: Ensure the site works correctly in different browsers and screen sizes.
5. **Commit and Push**: `git commit -m "Add your message"` and `git push origin feature/your-feature-name`
6. **Submit a Pull Request**: Describe your changes and why they're beneficial.

- Ensure responsiveness across devices.
- Test for accessibility (e.g., alt text for images, keyboard navigation).
- Keep code clean and commented where necessary.
